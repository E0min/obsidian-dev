# 12주 로드맵

> TL;DR: 1~5주 FE 기본기(JS·브라우저·React), 6~7주 타입·성능·디버깅, 8~9주 네트워크·CS, 10~12주 프로젝트 소유권·AI-native·통합. 매주 개념 + 맨손 과제 + 면접 질문을 함께 한다.

주당 학습 시간은 시간 여유형(12주)을 가정한다. 각 주는 [[01-습관과-원칙]]의 주간 루틴(개념 → 맨손 → 디버깅 → 복습)을 따른다. 읽을 노트는 [[02-학습-인덱스]]에 영역별로 전부 정리돼 있다.

## 1주. JavaScript 코어 (1)

- **개념**: [[../javascript/실행-컨텍스트-메모리]](복습), [[../javascript/호이스팅-TDZ]], [[../javascript/클로저]]
- **맨손**: AI 없이 디바운스·쓰로틀 함수 직접 구현. 클로저로 비공개 카운터 만들기.
- **면접 Q**: 클로저로 즉시 실행 함수 카운터를 만들고 동작을 설명하라. var와 let의 반복문 차이는?

## 2주. JavaScript 코어 (2)

- **개념**: [[../javascript/동기-비동기]](복습), [[../javascript/이벤트-루프]], [[../javascript/프로토타입-체인]], [[../javascript/얕은-깊은-복사]]
- **맨손**: `Promise.all` 직접 구현. 바닐라 JS 투두리스트(추가·삭제·완료, 로컬스토리지 저장).
- **면접 Q**: setTimeout과 Promise.then이 섞인 코드의 출력 순서를 예측하라. 얕은 복사로 생기는 버그 사례는?

## 3주. 브라우저 동작 원리

- **개념**: [[../browser/렌더링-파이프라인]], [[../browser/이벤트-위임-버블링]], [[../browser/CORS]], [[../browser/브라우저-저장소]]
- **맨손**: 이벤트 위임으로 동적 리스트 구현. Performance 탭으로 reflow 유발 코드와 transform 코드의 비용 비교 측정.
- **면접 Q**: 주소창에 URL을 입력하면 화면이 그려지기까지 무슨 일이 일어나는가? CORS는 왜 발생하고 누가 해결하는가?
- **프로젝트 연결**: xAB의 OAuth 콜백 X-Forwarded-Host 이슈 ([[03-프로젝트-소유권]])

## 4주. React 멘탈 모델 (1)

- **개념**: [[../react/리렌더링-트리거]], [[../react/reconciliation-key]], [[../react/React-memo]](복습)
- **맨손**: useState·useEffect만으로 작은 검색 필터 앱. React DevTools Profiler로 리렌더 측정, memo로 끊고 전후 비교.
- **면접 Q**: 리렌더를 일으키는 3가지는? 리스트 key에 배열 index를 쓰면 왜 위험한가?

## 5주. React 멘탈 모델 (2)

- **개념**: [[../react/lifecycle-useEffect]](복습), [[../react/strict-mode-이중호출]], [[../react/하이드레이션]]
- **맨손**: useEffect 클린업이 빠진 코드를 StrictMode로 재현하고 클린업으로 고치기.
- **면접 Q**: StrictMode는 왜 개발 모드에서 두 번 실행하는가? 하이드레이션 mismatch는 언제 나는가?
- **프로젝트 연결**: MindGraph의 StrictMode 비활성화와 D3+SSR 격리 ([[03-프로젝트-소유권]])

## 6주. TypeScript 심화

- **개념**: [[../typescript/타입-좁히기]](복습), [[../typescript/제네릭]], [[../typescript/유틸리티-타입]]
- **맨손**: `Partial`, `Pick`을 매핑된 타입으로 직접 구현. 본인 코드의 any를 제네릭으로 교체.
- **면접 Q**: 제네릭과 any의 차이는? Pick과 Omit을 언제 쓰는가?

## 7주. 성능과 디버깅

- **개념**: [[../performance/core-web-vitals/웹-성능-측정]](Core Web Vitals), [[../debugging/크롬-devtools-디버깅]]
- **맨손**: 본인 프로젝트 한 페이지를 Lighthouse로 측정 → LCP·CLS·INP 개선 → 재측정.
- **면접 Q**: LCP를 개선하는 방법은? 번들 크기를 줄이는 수단은?
- **프로젝트 연결**: chatGraph 843→355줄 리팩토링에 리렌더·번들 수치 근거 보강.

## 8주. 네트워크 기초

- **개념**: [[../../CS/network/http-https]](HTTP·상태코드), [[../../CS/network/쿠키-세션-jwt]](인증), [[../browser/브라우저-저장소]](토큰 보관)
- **맨손**: fetch로 공개 API를 직접 호출하며 로딩·에러·재시도·캐시를 손으로 처리.
- **면접 Q**: GET과 POST의 차이, 멱등성이란? 401과 403의 차이는? JWT를 어디에 보관하고 왜?

## 9주. CS 기초 (자료구조·알고리즘)

- **개념**: [[../../CS/_MOC]], [[../../CS/data-structures/hash-table]]
- **맨손**: [[../../CodingTest/_MOC]] 주 5문제. 시간·공간 복잡도를 매번 적기.
- **면접 Q**: 해시맵과 배열의 탐색 복잡도 차이는? 본인이 푼 문제의 시간 복잡도를 설명하라.

## 10주. 프로젝트 소유권

- **개념/작업**: [[03-프로젝트-소유권]] 워크시트로 Fit·chatGraph·xAB·MindGraph의 모든 기술 결정 재유도.
- **맨손**: 각 프로젝트의 핵심 코드 1개를 닫아두고 기억으로 재현.
- **면접 Q**: [[../../취업/interviews/_MOC]]의 프로젝트별 예상 질문 전부에 소리 내어 답하기.

## 11주. AI-native 체화

- **개념/작업**: [[04-ai-native-엔지니어]]의 검증·명세·분해 실습. AI를 튜터로 쓰는 루틴 정착.
- **맨손**: spec-first로 작은 사이드 프로젝트 명세서 작성 후 구현 시작.
- **연결**: [[../../AI-Native/spec-driven/sdd-프레임워크]], [[../../AI-Native/_MOC]]

## 12주. 통합

- **맨손**: 사이드 프로젝트를 AI 최소로 처음부터 완성. 막히면 가설 → 문서 → AI 순서.
- **작업**: 모의면접 반복. 답이 막히는 주제를 [[02-학습-인덱스]]에서 다시 학습.
- **점검**: [[../_MOC]] dataview에서 confidence 1~2 노트가 남았는지 확인하고 보강.
