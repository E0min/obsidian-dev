# 프로젝트 소유권

> TL;DR: 이력서의 프로젝트 4개를 면접에서 방어하려면 모든 기술 결정을 스스로 다시 유도할 수 있어야 한다. 각 결정에 "왜 이걸 골랐고, 대안은 무엇이며, 실패 케이스는 어떻게 처리했는가"를 한 문장씩 직접 적는다.

쓰는 법: 각 프로젝트의 결정마다 빈칸을 소리 내어 답하고, 막히면 해당 개념 노트로 돌아가 학습한 뒤 다시 답한다. 면접 자료는 [[../취업/interviews/_MOC]]에 프로젝트별로 정리돼 있으니 같이 갱신한다.

## Fit (WebRTC 컨퍼런스)

면접 자료: [[../취업/interviews/projects/fit]] · 개념: [[../CS/network/webrtc]], [[../FE/debugging/크롬-devtools-디버깅]]

이력서 문장: setRemoteDescription 직후 일괄 flush하는 버퍼링 로직으로 PeerConnection 성공률을 높임.

재유도 질문:
- ICE candidate를 왜 큐에 버퍼링했는가. setRemoteDescription 전에 도착한 candidate를 바로 addIceCandidate 하면 무엇이 깨지는가
- TURN 서버 추가나 trickle ICE 설정 같은 대안 대비 이 방식을 고른 이유는
- 연결 실패율을 어떻게 측정했는가. "성공률을 높였다"의 전후 수치는
- 연결이 끝내 실패할 때 사용자에게 무엇을 보여줬는가

## chatGraph (대화형 데이터 시각화)

면접 자료: [[../취업/interviews/projects/chatgraph]] · 개념: [[../FE/react/리렌더링-트리거]], [[../FE/javascript/클로저]], [[../FE/javascript/이벤트-루프]]

이력서 문장: 843줄 use-question-tree 훅을 책임 단위로 분해해 355줄로 축소. temp ID로 즉시 라우팅 후 mutation 성공 시 실제 ID로 교체하는 optimistic 라우팅.

재유도 질문:
- 843줄 훅을 어떤 책임 경계로 쪼갰는가. 분해 후 리렌더 범위나 테스트 용이성은 어떻게 달라졌는가
- 줄 수 외에 개선을 보여줄 수치(리렌더 횟수, 번들 크기, 함수 복잡도)는 무엇인가
- optimistic 라우팅에서 mutation이 실패하면 temp ID 경로를 어떻게 롤백했는가. 사용자 경험은
- temp ID와 실제 ID 교체 시점에 진행 중이던 다른 요청은 어떻게 처리했는가

## xAB (A/B 테스트 SNS)

면접 자료: [[../취업/interviews/projects/xab]] · 개념: [[../FE/browser/CORS]], [[../FE/browser/이벤트-위임-버블링]], [[../FE/react/reconciliation-key]]

이력서 문장: Intersection Observer로 페이지 분할 호출과 지연 로딩이 적용된 무한 스크롤. 로드밸런서 환경의 OAuth 콜백 오리진 오류를 X-Forwarded-Host 헤더 우선 참조로 복원.

재유도 질문:
- 무한 스크롤 리스트의 key를 무엇으로 잡았는가. index를 썼다면 어떤 문제가 생기는가
- 페이지 분할 호출에서 중복 요청이나 마지막 페이지를 어떻게 막았는가
- 로드밸런서 뒤에서 OAuth 콜백 오리진이 왜 어긋났는가. X-Forwarded-Host로 원래 출처를 어떻게 복원했는가
- 이건 CORS 문제인가 리다이렉트 URI 불일치 문제인가. 둘을 구분해 설명할 수 있는가

## MindGraph (Cytoscape 그래프, 1인 개발)

면접 자료: (없음, 추가 필요) · 개념: [[../FE/react/하이드레이션]], [[../FE/browser/렌더링-파이프라인]]

이력서 문장: 직접 구현한 D3 force 그래프 렌더러를 Cytoscape.js(fcose·dagre)로 빅뱅 이관하고 트리 뷰는 React Flow로 분리(ADR-0010).

재유도 질문:
- 직접 만든 D3 force 엔진을 버리고 Cytoscape로 이관한 이유는. 유지 비용과 제품 방향(플랫 그래프 + 트리 분리)에 어떻게 연결되는가
- Cytoscape는 Canvas, D3는 SVG로 렌더한다. 노드 수가 늘 때 두 방식의 렌더 비용 차이는
- 빅뱅 이관 중 미니맵·마퀴 선택이 일시 미구현인데, 빅뱅과 점진 이관 중 왜 빅뱅을 택했는가
- lib/engine/shared의 transformToGraphData를 재사용한 이유는. 데이터 계층과 렌더 계층 분리가 이관을 어떻게 도왔는가
- Cytoscape도 클라이언트 전용(Canvas·document 필요)인데 SSR 경계는 어떻게 처리했는가
- 1인·출시 전 프로젝트인데 사용자 가치나 검증을 어떻게 보여줄 것인가

## 공통 마무리

- 4개 프로젝트의 핵심 코드 1개씩을 닫아두고 기억으로 재현한다.
- 각 재유도 질문에 답한 내용을 [[../취업/interviews/projects/fit]] 등 해당 면접 노트에 옮겨 면접 자료를 보강한다.
- MindGraph 면접 노트는 새로 만들어 위 답을 정리한다.
