# Sensory — 감각 아카이브 기획안

> 영화/음악을 감각적 오브제로 박제하는 개인 큐레이션 아카이브.
> 인스타 스토리 공유까지 고려한 "취향 원페이저" 바이럴 서비스.

---

## 1. 프로젝트 개요

- **프로젝트명**: Sensory (센서리)
- **슬로건**: "기록이 예술이 되는 개인 전시 공간"
- **컨셉**: 영화·음악을 단순 데이터가 아닌 시각적·청각적 텍스처(Texture)를 가진 오브제로 변환해 전시하는 커스텀 아카이빙 플랫폼
- **타겟**: 자신의 취향을 시각적으로 소장하고 SNS로 자랑하고 싶어하는 시네필/헤비 리스너
- **차별화**: 리스트/평점 중심의 기존 앱(왓챈/리브레)과 달리, 색감·태그·오브제 형태로 "감각 자체"를 박제

---

## 0. North Star — 시작=아카이브, 방향=디깅 (2026-06-21 결정)

Sensory는 두 층이 플라이휠로 맞물린 하나의 제품.

- **아카이브 = 획득·데이터 층 (시작/입구)**: 유저가 들어오는 입구. 디깅이 작동할 연료를 채움. 유저는 이 비용을 "예쁘게 박제하고 자랑하는 재미"로 기꺼이 지불. 바이럴(공유 카드)이 성장 엔진.
- **디깅 = 유지·가치 층 (방향/목적지)**: 유저가 재방문하는 이유. 차별화 해자. 늦게 나오지만 진짜 목적지. "남의/세상의 취향을 파고들어 새로 발견".

### 플라이휠 (성장 루프)

```
박제(예쁨) → 공유 카드(자랑) → 신규 유입 + 데이터 시드
   ↑                                      ↓
재방문(디깅이 재밌어서) ← 디깅 풍부해짐 ← 데이터·사람 차오름
```

### 이 결정의 3대 함의

1. **자랑의 정의 전환**: 공유 카드 = 취향의 *넓이*("뭘 봤다") → 취향의 *깊이*("이 감독 계보를 여기까지 팠다") 자랑. 깊이를 자랑하는 순간 소셜(자랑)과 몰입(디깅)이 하나로 붙음. 공유 카드 = 디깅의 결과물.
2. **데이터 모델 선제 결정 (⭐ 엔지니어링 핵심)**: 디깅은 Phase 3지만, 그걸 가능케 하는 **노드/엣지 데이터 모델은 Phase 1에 깔아야** 함. MVP에서 아이템을 평면 리스트로 저장하면 나중에 계보 그래프 만들 때 전체 마이그레이션 발생. → §8 스키마에 노드/엣지 선반영. MindGraph 노드 그래프 시각화 자산이 디깅 엔진 뼈대로 재사용.
3. **디깅 콜드 스타트 = 공개 데이터로 해결**: UGC 차기 전에도 디깅이 살아있어 보여야 함. TMDB(비슷한 영화/감독), Spotify related artists, WhoSampled(샘플 DB)로 공개 그래프 선탑재 → 유저 1명만 있어도 "Drive→레픈→신스웨이브" 계보 펼쳐짐. 사람 데이터 차면 그 위에 "닮은 사람" 층 얹힘.

### 핵심 질문 답 (방향 확정)

> 메인 루프가 "내 취향을 예쁘게 박제해 자랑"(아카이브)이냐 "남의/세상 취향을 파고들어 발견"(디깅)이냐?
> → **시작은 아카이브, 방향은 디깅.** 기록을 입구로, 디깅을 목적지로 단계 설계.

## 2. 핵심 디자인 컨셉: Adaptive Aesthetic

사용자가 3가지 마스터 테마를 선택하고 그 위에 세부 텍스처를 커스텀.

| 테마명 | 시각적 특징 | 주요 인터랙션 |
|--------|------------|--------------|
| **Analog Vinyl** | 빛바랜 종이, LP판 회전, 티켓 스텁 | 클릭 시 지직거리는 노이즈 사운드, 페이지 전환 시 페이드 |
| **Digital Glitch** | 네온 컬러, 스캔라인, 데이터 칩, 그리드 | 마우스 오버 시 글리치 효과, 시스템 폰트 |
| **Ethereal Flow** | 유리 질감(Glassmorphism), 그라디언트, 부유 오브제 | 마우스 움직임에 따라 유동적 반응 (중력/관성) |

## 3. 주요 기능

### A. 감각적 업로드 (Sensory Input)
- TMDB/Spotify/YouTube API 연동 검색
- 업로드 이미지에서 주요 5가지 컬러 자동 추출 → Primary Color로 지정
- 형용사 중심 태깅 시스템 (#서늘한 #눅눅한 #찬란한 #심장을울리는)

### B. 모듈형 커스텀 전시 (Custom Gallery)
- **오브제 선택**: LP판 / 필름 스트립 / 폴라로이드 / 영화 티켓 / 데이터 칩 / 추상 기하 도형
- **Free Canvas**: 그리드 없는 자유 배치, 크기 조절 자유
- **Texture Overlay**: Film Grain / VHS Noise / Glassmorphism / Scanline
- **레이아웃 물리법칙**: Pile(중력으로 쌓이기) / Orbit(우주처럼 퍼지기) / Flow(시간 강물처럼 흐르기)

### C. 감상 모드 (Immersive Player)
- **Sync Play**: 음악 기록 열람 시 15초 하이라이트 + 시각적 이퀄라이저
- **Full-Screen Theater**: 영화 기록 클릭 시 암전, 대사/스틸컷만 노출
- **타임스탬프 메모**: 음악 특정 구간(예: 1분 20초 드롭)에 감상 박제

### D. 공유 및 바이럴 (Viral Hook)
- 자신의 커스텀 테마 그대로 적용된 **Read-only 공유 URL** 생성
- 인스타 스토리 최적화 사이즈 (1080×1920) 자동 캡처 + 다운로드
- 월간 감각 리포트 → "당신의 5월은 ##우울한 색 ##XX 장르였습니다" 카드 생성 (스포티파이 랩드 스타일)

## 4. 하이브리드 데이터 저장 전략

### Guest Mode (비로그인)
- 웹: **IndexedDB** (대용량 이미지 캐싱), **localStorage** (설정값)
- 모바일 웹: localStorage / IndexedDB 동일 사용 가능
- 추후 RN 앱: AsyncStorage
- 즉시 시작 가능, 진입 장벽 0

### Cloud Sync (로그인 후)
- Supabase Auth 로그인 시 로컬 → DB upsert 마이그레이션
- 기기 간 동기화, 공유 URL 활성화
- "클라우드에 저장하고 어디서든 확인하기" CTA로 자연스러운 로그인 유도

## 6. 모바일 최적화 전략

- **Adaptive Performance (LOD)**: 저사양 기기 감지 시 셰이더 효과 비활성화/정적 이미지 대체
- **Touch-First UX**: Framer Motion drag 옵션으로 터치 최적화, 핀치 줌 지원
- **Lazy Loading**: 캔버스 밖 오브제 미렌더링 (Intersection Observer)
- **Responsive Physics**: 화면 크기에 따라 오브제 크기 자동 스케일링
- **PWA 설정**: "홈 화면 추가" 지원 + 오프라인 캐시
- **Image Optimization**: next/image AVIF/WebP 자동 변환, srcset 디바이스별 제공

## 8. Supabase 테이블 설계 (초안)

```sql
-- users (Supabase Auth 기본)
profiles (
  id uuid pk references auth.users,
  username text unique,
  theme jsonb,  -- 마스터 테마 + 텍스처 설정
  created_at timestamptz
)

archives (
  id uuid pk,
  user_id uuid references profiles,
  type text,  -- 'movie' | 'music' | 'album'
  title text,
  metadata jsonb,  -- TMDB/Spotify 원본
  cover_url text,
  palette text[],  -- 추출된 5 컬러
  tags text[],     -- 형용사 태그
  note text,       -- 한줄 감상
  timestamp_seconds int,  -- 음악일 때 타임스탬프
  object_type text,  -- 'vinyl' | 'ticket' | 'polaroid' | 'chip'
  position jsonb,    -- {x, y, scale, rotate}
  created_at timestamptz
)

galleries (
  id uuid pk,
  user_id uuid references profiles,
  slug text unique,  -- 공유용 URL
  is_public bool,
  layout jsonb,      -- 캔버스 전체 레이아웃
  created_at timestamptz
)
```

## 12. 취향 임베딩 & 유사 취향 분포 (Taste Embedding)

> "당신과 닮은 취향의 사람들은 이 은하의 이 자리에 있습니다" — 분포 시각화 카드가 바이럴 훅.
> 기술적으로 충분히 가능. 영화/음악은 벡터화하기 좋은 데이터라 궁합이 좋음.

### ⚠️ Spotify API 제약 (2024-11-27 deprecation) — 핵심 정정

Spotify는 2024-11-27부로 **신규 앱**에 `audio-features`, `audio-analysis`, `related-artists`, `recommendations`, `preview_url` 차단. 그날 이전 quota extension 보유 앱만 예외. **Sensory(2026 신규 앱)는 전부 못 씀.** → 원래 "Spotify audio features 공짜 벡터" 전제 폐기. 아래 대체 스택으로 전환.
출처: developer.spotify.com/blog/2024-11-27-changes-to-the-web-api

**음악 데이터 대체 스택 (대부분 오픈/무료, rug-pull 위험 낮음)**

| 역할 | 원래(막힘) | 대체(열림) |
|------|-----------|-----------|
| 검색·메타·앨범아트·로그인·청취기록 임포트 | Spotify | **Spotify ✓ 여전히 가능** (온보딩 채널로) |
| 오디오 특성 벡터 | Spotify audio-features ✗ | **AcousticBrainz** 데이터셋(Essentia 계산, 수백만 곡, 다운로드) |
| 유사·계보 그래프 | Spotify related-artists ✗ | **Last.fm** artist.getSimilar + **MusicBrainz** 관계(영향·멤버·소속) |
| 추천·딥컷 | Spotify recommendations ✗ | **ListenBrainz** 추천 / 자체 임베딩 kNN |
| 미리듣기(15초 하이라이트) | Spotify preview_url ✗ | Deezer / YouTube / Apple 프리뷰 |

### 12-1. 임베딩 재료 (대체 스택 반영)

| 재료 | 출처 | 비고 |
|------|------|------|
| **음악 오디오 특성** | AcousticBrainz 데이터셋 (Essentia) | danceability/mood/BPM 등. Spotify audio-features 대체. 2022년 수집 중단됐으나 백카탈로그 커버 |
| **유사·태그** | Last.fm (getSimilar, tags) + MusicBrainz 관계 | 계보 엣지 + 태그 임베딩 재료. 디깅에 더 적합 |
| **형용사 태그** | OpenAI `text-embedding-3-small` ($0.02/1M tok) 또는 로컬 `bge-m3`(무료) | #서늘한 #눅눅한 + Last.fm 태그 → 텍스트 임베딩 |
| **컬러 팔레트** | 앨범 아트 추출 5색 → LAB 색공간 벡터 | 추가 비용 0 |
| **영화 장르/키워드** (Phase 2+) | TMDB | 영화 합류 시 |

### 12-2. 사람 1명 = 벡터 1개로 압축

- **MVP**: 아카이브 내 아이템 벡터들의 평균 풀링(mean pooling) → 사용자 1명 = 벡터 1개. 단순하지만 잘 작동
- **고도화**: 최근 기록 가중치 / 취향 다극(호러 영화 + 시티팝)인 사람을 위해 아이템을 2~3개 클러스터로 나눠 멀티 벡터

### 12-3. 유사도 검색 + 분포 시각화

- **유사도**: Supabase **pgvector** 익스텐션 정식 지원. 벡터 컬럼 + 코사인 유사도 HNSW 인덱스 → "나와 가까운 사람 N명" SQL 한 방
- **분포 비주얼**: 전체 사용자 벡터(수십~수백 차원)를 **UMAP/t-SNE/PCA**로 2D 좌표 투영 → 산점도. 내 점은 크게, 닮은 사람이 성단처럼 모이는 그림 = 인스타 공유 카드의 핵심

### 12-4. 현실적 함정

- **콜드 스타트**: 사용자 적으면 분포가 안 그려짐 → 초기엔 시드 페르소나 군집 20개를 미리 깔거나, "사람 분포"보다 **콘텐츠(곡/영화) 군집**을 먼저 출시 (데이터가 빨리 참)
- **차원축소 재계산 비용**: UMAP은 데이터 변경 시 전체 재학습이라 무거움 → 실시간 X, **하루 1회 배치** 갱신 + 신규 유저는 기존 맵에 근사 투영(`transform`)
- **프라이버시**: 남의 취향 좌표 노출은 민감 → 익명 점(닉네임/아바타만), 공개 동의한 사용자만 분포 포함

### 12-5. 난이도 평가

| 항목 | 난이도 | 비고 |
|------|--------|------|
| 음악 취향 벡터 | 🟢 낮음 | Spotify가 벡터 제공 |
| 태그/색 임베딩 | 🟢 낮음 | 임베딩 API 호출만 |
| pgvector 유사도 검색 | 🟢 낮음 | Supabase 기본 지원 |
| 2D 분포 시각화 | 🟡 중간 | UMAP 배치 + 프론트 산점도 |
| 콜드 스타트 해결 | 🟡 중간 | 시드 데이터 전략 필요 |

→ **결론**: MVP 이후 Phase 3 바이럴 훅으로 붙이기 적합. 월간 감각 리포트와 묶어 "내 취향 좌표 + 닮은 사람 분포" 카드로 내보내면 공유 동기 강화.

## 14. 디깅 엔진 (방향/목적지 층, 2026-06-21)

> "디깅 모멘텀"(트렌드 코리아 키워드, 크레이트 디깅 어원) = 알고리즘이 떠먹여주는 추천에 질린 유저가 직접 파고드는 경험. 영화·음악과 정서 일치.

### 14-1. 디깅 메커니즘 4종

1. **취향 계보(lineage) 그래프** — Drive→감독 레픈→신스웨이브→Kavinsky→모로더. 한 작품에서 가지 쳐 파고드는 노드 그래프. MindGraph 그래프 시각화 자산 재사용.
2. **레이어드 디깅** — 한 작품에서 한 칸 더. 영화 OST→작곡가→영향받은 앨범. 곡 샘플→원곡→그 시대. "한 칸 더 들어가기" 버튼.
3. **사람을 타고 디깅** — 알고리즘 X. 취향 벡터(§12) 가까운 사람의 아카이브로 들어가 발굴. "이 사람 취향 좋다 → 이 사람 딥컷 구경."
4. **딥컷 우선** — 메인스트림 제외, 깊은 곳 추천. "알 만한 건 다 알아 → 모를 만한 걸 보여줄게."

### 14-2. 콜드 스타트 해결 (공개 데이터 선탑재)

UGC 차기 전에도 디깅이 살아있어 보이게 (음악-우선, Spotify 대체 스택 반영):
- **Last.fm** — artist.getSimilar / track.getSimilar (유사 엣지)
- **MusicBrainz** — 아티스트 관계(영향받음·멤버·소속) → 계보 엣지가 공개 데이터로 공짜
- **ListenBrainz** — 추천 / 딥컷
- (Phase 2+ 영화 합류) **TMDB** — 비슷한 영화 / 감독 필모 / 키워드, **WhoSampled** — 곡 샘플 계보
→ 유저 1명만 있어도 계보 그래프 펼쳐짐. 사람 데이터 차면 "닮은 사람" 층 얹힘.

### 14-3. 데이터 모델 함의 (⭐ Phase 1 선제 결정)

디깅은 Phase 3~4지만 **노드/엣지 스키마는 MVP에 깔아야** 마이그레이션 회피.

```sql
-- archives = 노드. 평면 리스트 금지, 그래프 노드로 설계
-- 엣지 테이블 (계보/연결)
edges (
  id uuid pk,
  from_node uuid references archives,  -- 또는 public_nodes
  to_node uuid,
  relation text,   -- 'director'|'sampled'|'ost'|'similar'|'influenced_by'
  source text,     -- 'tmdb'|'spotify'|'whosampled'|'user'
  weight float
)
-- public_nodes: UGC 아닌 공개 그래프 노드 (영화/감독/아티스트/곡)
```
