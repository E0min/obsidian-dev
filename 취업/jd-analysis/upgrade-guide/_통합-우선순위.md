# mindgraph 업그레이드 — 4직무 통합 우선순위

> 4개 직무 가이드([[FDE]]·[[Product-Engineer]]·[[AI-Native-Builder]]·[[FE]])의 업그레이드 후보를 직무 겹침과 난이도로 다시 정렬한 인덱스. "다음에 뭐부터" 한눈에 보는 용도.

## 1. 큰 그림 — 같은 코드, 직무별로 다른 면

| 직무 | mindgraph에서 비추는 면 | 지금 가장 강한 자산 |
|------|------------------------|---------------------|
| AI-Native | `.claude/` 하네스 자체 (9훅·3phase·7부서9에이전트·context-map) | AI 운영 체계를 직접 설계·운영 |
| Product Eng | 1인 PO가 가설→출시→지표로 끌고 간 흐름 | 풀스택 + 4중 SSOT + 관측성 사전구축 |
| FDE | 고객 배포·응대까지 혼자 끝내는 end-to-end | 출시 후 응대 인프라(PostHog/Sentry/OTel) 사전배치 |
| Frontend | React19/Next16·D3 SSR 격리·Vite 3종·Tiptap 깊이 | 빌드·렌더링 아키텍처 |

## 2. 직무별 핵심 업그레이드 (각 가이드 우선순위 압축)

### AI-Native Builder — 증명: AI 워크플로우를 직접 만든 빌더
1. LLM 호출 관측성 (OTel GenAI/Langfuse로 model·토큰·비용)
2. 검색 eval (recall@k·MRR) — ※임베딩 추천 미구현이라 임베딩 구현이 선결
3. 하네스 외부 자산화 (`.claude/skills`+`hooks`+`context-map` → npm/별도 repo)

### Product Engineer — 증명: 정의→지표로 검증하는 제품 사고
1. PostHog A/B 분기 + funnel (방문→캡처→트리→재방문)
2. Core Web Vitals 자동 측정 (LCP·INP·CLS)
3. 하네스 sub-agent 추출 (AIN과 동일 자산, PE 시그니처)

### FDE — 증명: 고객 배포·운영을 혼자 끝냄
1. 오프라인-우선 E2E (Playwright 네트워크 차단 시나리오)
2. 비기술 온보딩 + 응대 화면 (Sentry feedback/api 모아보는 admin)
3. MCP 서버화 (`.mcp.json` 활용, Anthropic FDE 시그니처)
4. ⚠️ 최대 Gap: 외부 고객 1팀 도입 사례 — 룰 A상 실제 배포·온보딩 후에만 본문 기록 가능

### Frontend — 증명: 성능·접근성·아키텍처 깊이
1. 접근성 axe + Lighthouse CI (캡처/canvas/sidebar 위반 0)
2. Core Web Vitals + 임계 회귀 차단 (PE와 동일 작업)
3. Storybook + 시각 회귀 / 디자인 토큰화 / RSC 비중 확대

## 3. 통합 우선순위 (효율 순 — 하나로 여러 직무 커버)

| 순서 | 업그레이드 | 커버 직무 | 난이도 | 왜 먼저 |
|------|-----------|-----------|--------|---------|
| 1 | **측정 묶음**: Core Web Vitals + LLM 관측성 | PE·FE·FDE·AIN 전부 | S | 4직무 공통 최대 약점이 정량 측정. OTel/PostHog 이미 깔림. 한 PR로 전 직무 약점 해소 |
| 2 | Playwright E2E → GitHub Actions CI | FDE·PE·FE | S | Playwright 이미 있음, CI만 연결 |
| 3 | 접근성 axe + Lighthouse CI | FE(+PE 품질) | S | FE 시그널 가장 빠른 가시화 |
| 4 | **하네스 외부 자산화** (`.claude` → 패키지) | AIN·PE | M~L | 가장 차별적. 외부 설치·검증 가능 자산. 시간값어치 큼 |
| 5 | MCP 서버화 | FDE·AIN | M | `.mcp.json` 이미 있어 자연스러운 확장 |
| 6 | getmindgraph.com 실제 출시 + 사용자 1팀 | 전 직무, FDE Gap | L | "출시 전"→"운영 중"으로 바뀌는 순간 설득력 급상승 |

**가장 임팩트 큰 한 방**: 1번(측정)으로 4직무 공통 약점을 메우고, 4번(하네스 외부 추출)으로 AIN/PE 시그니처를 외부 검증 가능하게. 약점과 강점을 각각 정조준.

## 4. 룰 A 주의 (미구현 → 구현 후에만 본문)

임베딩 추천·검색 eval(recall@k)·H1/H3 정량 측정은 **현재 코드 미구현**(2026-06-03 확인, 본문 제거 완료). 위 업그레이드로 실제 구현한 뒤에만 이력서·포폴 본문에 반영. 지금은 업그레이드 후보로만 둠.
